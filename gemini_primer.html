Your Website: An Illustrated Primer

Part 1: How Your Website Currently Works

Think of your website like a beautiful, modern art gallery.

  • The Gallery Building (src/app): This is the main structure of your website. The src/app/page.tsx file is the grand entrance hallâ€”it's the first thing people see. The other files in src/app (like avatar/page.tsx, protocol/page.tsx, etc.) are the individual exhibition rooms. They are part of the main building and share its overall architecture.
  • The Artworks (public directory): This is where the actual content is stored. Each of your content pieces (like esg-content.html and protocol-content.html) is a finished, self-contained artwork. It has its own styling and presentation, separate from the gallery itself.
  • The Exhibition (<iframe>): To display an artwork, each exhibition room (avatar/page.tsx, etc.) has a large window (<iframe>) that looks directly at one of the artworks in the public directory. You're not bringing the artwork into the room; you're just showing it through a window.
  • The Storeroom (content directory): This directory is like a storeroom full of raw materials (your writings, ideas, etc.). Currently, it seems like this storeroom is not being used. The website is displaying the finished artworks from the public directory instead of creating new ones from the materials in content.

How It All Fits Together

  1. A visitor arrives at your main page (khayali.xyz).
  2. They see the beautiful entrance hall with links to the different "chambers" (Observatory, Protocol, etc.).
  3. They click on a link, say "Avatar."
  4. They are taken to the "Avatar" exhibition room (/avatar).
  5. This room shows them the esg-content.html artwork through a window (<iframe>).

How to Add or Change Things (The Current Way)

  • To change existing content: You need to find the corresponding HTML file in the public directory (e.g., public/avatar/esg-content.html) and edit the HTML code directly. This is like being a restorer who touches up a painting.
  • To add new content: You would need to:
    1. Create a brand new, self-contained HTML file with all its own styling.
    2. Put it in the appropriate folder inside the public directory.
    3. Create a new "exhibition room" (a new folder and page.tsx file in src/app) with an <iframe> pointing to your new HTML file.

Conclusion: This setup works, but it's rigid, hard to maintain, and leads to a disconnected look and feel.

Part 2: A Better Way - The Dynamic Content Engine

Instead of creating separate, self-contained HTML files, we can build a system that automatically takes your raw writings and displays them in a beautiful, consistent way across the entire site. This is how most modern content websites work.

The Big Idea

We'll use the content directory as the single source of truth. You'll write your content in a simple format called Markdown, and the website will automatically turn it into a webpage.

A Markdown file is just a plain text file. You can use simple symbols to format your text. For example:

  • # A Big Heading
  • ## A Smaller Heading
  • *A list item*
  • **Bold text**

At the top of each Markdown file, you can add a "frontmatter" section with key information, like a title and a summary. The website will use this information to create nice-looking index pages and previews.

How the New System Would Work

  1. You write: You create a new .md (Markdown) file in your content directory. For example, content/writings/my-new-poem.md.
  2. The website finds it: When a visitor comes to your site, the website will automatically scan the content directory to find all your Markdown files.
  3. The website displays it: The website will generate a new page for each Markdown file, using the content and frontmatter. It will apply the same beautiful, consistent styling to every page.

The Advantages

  • Easy to Use: No more fighting with HTML! Just write your text in a simple, clean format.
  • Consistent Design: All your content will automatically share the same beautiful design of your main site. No more disconnected styles.
  • Easy to Manage: All your content lives in one place (the content directory). Adding, editing, and removing content is as simple as managing text files.
  • Future-Proof: This is a standard, robust way to build content-heavy websites. It will be much easier to add new features in the future.
